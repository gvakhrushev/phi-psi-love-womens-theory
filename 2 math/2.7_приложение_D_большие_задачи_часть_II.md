## D1. Kakeya/Besicovitch: минимальная размерность без адресации
**Классика:** множество \(E\subset\mathbb R^2\), содержащее отрезок **любой** ориентации, должно иметь минимально возможную размерность (в плоскости — 2?).

### D1.1. Встраивание в T_all
- **Сцена:** непрерывный поворот отрезка — чистая **ψ‑голономия** (инверсия/ориентация) **внутри**. Любая «адресация угла» — внешний ярлык ⇒ \(\Delta K>0\).

### D1.2. Инвариант покрытия (min‑K)
Определим «стоимость безметочного покрытия» \(\mathcal K(E)\):
\[ \mathcal K(E):= \inf_{\mathcal F\in\mathfrak C(E)} K(\mathcal F), \]
где \(\mathfrak C(E)\) — семейство **безметочных** покрытий всем набором направлений; «безметочность» = отсутствие внешнего кода ориентаций.

### D1.3. Лемма (порог размерности)
Если \(E\) допускает полный непрерывный пучок направлений **без** адресной разметки, то
\[ \dim(E) \ge d_\min \quad (\text{в }\mathbb R^2: d_\min=2). \]
*Идея:* попытка опустить \(\dim(E)\) < 2 требует «адресовать» повороты (иначе пучок теряется) ⇒ внешний бит ⇒ \(\Delta K>0\).

### D1.4. Редукция (скелет)
1) Моделируем пучок направлений как ψ‑орбиты на \(G_{\phi,\psi}\).  
2) Безметочное покрытие ⇔ ψ‑перестановки реализуются внутренне.  
3) \(\dim<2\) ⇒ нужен внешний код углов ⇒ \(\Delta K>0\).  
4) Следовательно, **порог**: \(\dim(E)\ge2\) при полном пучке.

### D1.5. Фальсификация
- Построить \(E\) с полным набором направлений и \(\dim(E)<2\), **не** вводя адресации и при этом с **уменьшением** \(K\) (↓K).

### D1.6. Мини‑проверки
- Игрушечные решётки с ψ‑инвариантной перестановкой направлений; проверять рост \(K\) при попытке адресовать углы.  
- Сравнение покрытий: с адресацией vs без — оценка \(\Delta K\).

### D1.7. Псевдокод (аудит Kakeya)
```
Input: кандидат E ⊂ R^2
1) Синтезируй безметочное покрытие F (все направления)
2) Проверь, что перестановки направлений = ψ-операции внутри кода
3) Оцени K(F) и попытайся построить E' с dim(E')<2 без меток
4) fail, если нужные повороты требуют адресации (ΔK>0)
accept ⇔ полный пучок с dim(E)≥2 без ↑K
```

## D2. Проблема Лемера / Мера Махлера: ψ‑порог высоты
**Классика:** существует ли положительный нижний предел \(>1\) у меры Махлера \(M(\alpha)\) для алгебраических \(\alpha\), не являющихся корнями единства?

### D2.1. Встраивание в T_all
- **Сцена (высота=стоимость):** «высота» числа — описательная стоимость кода; ψ‑устойчивый порог — естественный min‑K барьер.

### D2.2. Инвариант φ‑ψ‑порога
Определим ψ‑пороговую величину \(H_{\min}\):
\[ H_{\min} := \inf_{\alpha\notin \mu_\infty} K_\mathrm{arith}(\alpha) \quad \text{(при ψ‑устойчивых преобразованиях внутри класса)}. \]
**Принцип:** \(\exists\,c_\phi>0\) такой, что \(H_{\min}\ge c_\phi\).

### D2.3. Лемма (устойчивость порога)
Любая последовательность \(\alpha_n\) с \(M(\alpha_n)\downarrow1\) без обращения к корням единства потребует либо (i) внешних ярлыков для кодирования «почти‑единичности», либо (ii) разрыва ψ‑устойчивости. В обоих случаях \(\Delta K>0\).

### D2.4. Редукция (скелет)
1) Интерпретировать высоту как стоимость минимального полинома (алфавит порождений фиксирован).  
2) Порог \(c_\phi\) — следствие невозможности «бесплатно» приблизить спектр к корням единства **внутри** без ярлыков.  
3) Любой снос порога без ярлыков уменьшал бы \(K\) ниже минимума — запрещено.

### D2.5. Фальсификация
- Построить \(\alpha_n\notin\mu_\infty\) с \(M(\alpha_n)\downarrow1\) **без** внешних ярлыков и с **снижением** общей описательной длины (↓K) при TR=1.

### D2.6. Мини‑проверки
- Анализ известных малых высот: где возникает необходимость внешнего бита в коде минимального полинома.  
- Вариации ψ‑операций (инверсия, сопряжения) на устойчивость \(K\).

### D2.7. Псевдокод (аудит высоты)
```
Input: список алгебраических α (не корни единства)
For α in list:
  encode α via minimal polynomial in fixed generator alphabet
  test ψ-stable transforms preserve K up to o(1)
fail, если α_n → 1 по M(α_n) и при этом K не растёт (ΔK≤0)
accept ⇔ существует c_φ>0: H_min ≥ c_φ
```

## D3. Гипотеза Литтлвуда: ψ‑сшивка диофантовых резонансов
**Классика:** \(\liminf_{n\to\infty} n\,\|n\alpha\|\,\|n\beta\| = 0\) — открыто в общем случае.

### D3.1. Встраивание в T_all
- **Сцена:** резонанс двух частот = ψ‑сшивка траекторий на φ‑решётке. Безметочная сшивка должна работать **внутри** кода.

### D3.2. Резонансный функционал (min‑K)
\[ R(\alpha,\beta) := \inf_{n\ge1} n\,\|n\alpha\|\,\|n\beta\|. \]
Если ψ‑сшивка реализуется **внутри** без адресации резонансов, то \(R=0\) (последовательность \(n_k\) находится «бесплатно»). Нужда в адресации ⇒ \(\Delta K>0\).

### D3.3. Лемма (безметочная сшивка ⇒ нулевой лиминф)
При ψ‑устойчивой сшивке пар \((\alpha,\beta)\) существует бесконечная последовательность \(n_k\) с \(R=0\). Отсутствие такой последовательности без адресации фиксирует положительный нижний барьер \(\varepsilon>0\).

### D3.4. Редукция (скелет)
1) Сконструировать общий код для пар \((\alpha,\beta)\) без ярлыков резонансов.  
2) Если \(R>\varepsilon\) неизбежно, значит резонансы требуют адресной разметки ⇒ \(\Delta K>0\) — не min‑K.  
3) ψ‑сшивка внутри ⇒ \(R=0\).

### D3.5. Фальсификация
- Найти пары \((\alpha,\beta)\) с **доказуемым** \(\inf n\,\|n\alpha\|\,\|n\beta\|>\varepsilon>0\) при запрете адресации и **одновременно** показать, что код короче (↓K) и TR=1.

### D3.6. Мини‑проверки
- Эксперименты на badly approximable/линейно независимых над \(\mathbb Q\): оценка поведения \(R\) при ψ‑вариациях кода.  
- Тест «бесплатности» поиска \(n_k\) (без внешнего индекса резонансов).

### D3.7. Псевдокод (аудит резонансов)
```
Input: пары (α,β)
For each pair:
  try build ψ-stable code without labeling resonances
  search n with small n||nα||·||nβ|| using internal iteration only
  if requires addressing resonances → ΔK>0 (not min-K)
accept ⇔ exists infinite {n_k} with R≈0 без адресации
```

## D4. Теорема Эрдёша–Турана (суммы‑произведения): φ–ψ барьер адресации
**Классика (вариант сумм‑произведений):** для конечного множества \(A\subset\mathbb R\) множества \(A+A\) и \(A\cdot A\) не могут быть одновременно «слишком малыми» (сильные формы связаны с \(|A+A|+|A\cdot A|\)).

### D4.1. Встраивание в T_all
- **Сцена:** переход add↔mult — баланс φ/ψ. Любая «тонкая» структура, делающая оба набора малыми, вынуждает **адресные ярлыки** (шаблоны сумм/произв.) ⇒ \(\Delta K>0\).

### D4.2. Инвариант (баланс φ/ψ)
Определим стоимость двух кодов:
\[ B(A):=K_{\mathrm{add}}(A+A)+K_{\mathrm{mult}}(A\cdot A). \]
**min‑K‑барьер:** \(B(A)\) имеет нижнюю границу \(\ge C_\phi\,K(A)\) при честной ψ‑встройке.

### D4.3. Лемма (двойное сжатие ⇒ адресность)
Если одновременно \(|A+A|\) и \(|A\cdot A|\) малы относительно \(|A|\) при фиксированном коде \(A\), то шаблоны сумм/произв. требуют внешних маркеров ⇒ \(\Delta K>0\).

### D4.4. Редукция (скелет)
1) Кодировать \(A\) в общем φ–ψ‑алфавите.  
2) Оценить \(K\) для \(A+A\) и \(A\cdot A\) **без адресации**.  
3) Двойной снос невозможен без ярлыков ⇒ барьер на \(|A+A|+|A\cdot A|\).

### D4.5. Фальсификация
- Построить семейство \(A_n\) с \(|A_n+A_n|+|A_n\cdot A_n|\) сублинейным при **запрете** любых адресных ярлыков и **снижении** общей описательной длины.

### D4.6. Мини‑проверки
- Рандомизированные \(A\): скан \(B(A)\) и попытки искусственных «структур» без роста \(K\).  
- Сравнение с геометрич. прогрессиями/сумм. прогрессиями: где именно возникает необходимость меток.

### D4.7. Псевдокод
```
Input: конечное A⊂R
1) encode A in common φ–ψ alphabet
2) compute K_add(A+A), K_mult(A·A) without labels
3) if both small and ΔK=0 → falsify; else accept barrier
```

## D5. Теорема Сземереди (длинные арифм. прогрессии): ψ‑сшивка без адресации
**Классика:** любое множество натуральных положительной верхней плотности содержит арифметические прогрессии любой длины.

### D5.1. Встраивание в T_all
- **Сцена:** прогрессия длины \(k\) — ψ‑сшивка \(k\) точек единой «фазы». Внешняя адресация шагов прогрессии — ярлык ⇒ \(\Delta K>0\).

### D5.2. Инвариант (φ/ψ‑регулярность)
Определим \(R_k(S)\) — минимальную стоимость кода, делающего \(k\)-AP **без** внешней адресации шагов. **min‑K:** при положительной плотности \(R_k\) ограничен сверху универсальной функцией от плотности; иначе требуются ярлыки.

### D5.3. Лемма (плотность ⇒ внутренняя регулярность)
Если верхняя плотность множества \(S\subset\mathbb N\) положительна, то существование \(k\)-AP достигается ψ‑сшивкой **внутри** без меток. Запрет меток и отсутствие AP привёл бы к росту \(K\).

### D5.4. Редукция (скелет)
1) Представить код \(S\) в φ–ψ‑алфавите.  
2) Показать, что попытка устранить \(k\)-AP требует адресации шагов ⇒ \(\Delta K>0\).  
3) Следовательно, при положительной плотности \(k\)-AP существуют без ярлыков.

### D5.5. Фальсификация
- Конструировать \(S\) положительной верхней плотности без \(k\)-AP, **не** вводя адресных ярлыков и со **снижением** общей описательной длины.

### D5.6. Мини‑проверки
- Стохастические модели (Бернулли с плотностью \(\delta>0\)): оценить \(R_k\) и роль ψ‑сшивки.  
- Псевдослучайные конструкции: где возникает необходимость внешних меток.

### D5.7. Псевдокод
```
Input: S⊂N, upper density δ>0
1) encode S in φ–ψ
2) try build k-AP internally without labels
3) if impossible without labels and K↓ → falsify; else accept
```

## D6. Теорема Эрдёша–Гинзбурга–Зива (EGZ): φ–ψ баланс сумм по модулю n
**Классика:** для любого \(A\subset\mathbb Z_n\) с \(|A|\ge 2n-1\) существует \(n\)-подмножество, сумма которого \(\equiv 0\pmod n\).

### D6.1. Встраивание в T_all
- **Сцена:** сумма по модулю — **ψ‑фазировка** (обратимость классов) внутри; вынос «индексов» подмножеств во внешний ярлык нарушает min‑K.

### D6.2. Инвариант (φ/ψ‑спектральная равномерность)
Определим стоимость равномерности классов:
\[ U_n(A):=K\big(\{\#[A\cap c]\}_{c\in\mathbb Z_n}\big). \]
**min‑K:** при \(|A|\ge2n-1\) безадресная фазировка обеспечивает нулевую сумму для некоторого \(n\)-набора.

### D6.3. Лемма (фазировка ⇒ нулевая сумма)
Если ψ‑фазы классов реализуются **внутри** без адресных меток, то существует \(n\)-поднабор с суммой \(0\pmod n\). Запрет ψ или адресация классов ⇒ \(\Delta K>0\).

### D6.4. Фальсификация
- Построить \(A\) при \(|A|\ge 2n-1\), где без ярлыков не удаётся получить нулевую сумму и при этом общий код **короче** (↓K).

### D6.5. Мини‑проверки
- Рандомизированные \(A\) в \(\mathbb Z_n\): оценка \(U_n(A)\) и нахождение нулевой суммы внутренним перебором фаз.

### D6.6. Псевдокод
```
Input: A ⊂ Z_n, |A|≥2n-1
1) encode class counts {#[A∩c]} in φ–ψ
2) search n-subset with sum≡0 using internal phase pairing
3) fail if needs labels (ΔK>0) and K↓; else accept EGZ
```

## D7. Теорема Грина–Тао: бесконечно много простых с длинными AP (ψ‑сшивка в редких наборах)
**Классика:** множество простых содержит арифметические прогрессии любой длины.

### D7.1. Встраивание в T_all
- **Сцена:** редкое множество с псевдослучайностью; k‑AP — ψ‑сшивка «фаз» в подалгебрах. Внешняя адресация прогрессий — ярлык ⇒ \(\Delta K>0\).

### D7.2. Инвариант (регулярность/псевдослучайность)
Вводим функционал \(R_k(\mathcal P)\) — стоимость кода, делающего k‑AP внутри без ярлыков при стандартных регуляризациях (селекторы, взвешивания). **min‑K:** при псевдослучайности \(R_k\) ограничен.

### D7.3. Лемма (псевдослучайность ⇒ внутренняя k‑AP)
Достаточная регулярность (в смысле Грина–Тао) обеспечивает ψ‑сшивку k‑AP **внутри** без меток. Требование адресации индексов прогрессий повышает \(K\).

### D7.4. Фальсификация
- Модель псевдослучайных «простых», где k‑AP отсутствуют **без ярлыков** и при этом общий код короче.

### D7.5. Мини‑проверки
- Тесты мажоризации/корреляций; имитации взвешенных простых с поиском k‑AP внутренним обходом без адресации.

### D7.6. Псевдокод
```
Input: weighted primes model P_w
1) verify pseudo-randomness tests (internal)
2) search k-AP without labeling indices
3) if impossible with K↓ → falsify; else accept
```

## D8. Теоремы типа Турана (экстремальная комбинаторика): барьер адресности структур
**Классика:** ограничения на число рёбер/подструктур при запрете малых подграфов (например, \(K_{r+1}\)).

### D8.1. Встраивание в T_all
- **Сцена:** запрет подструктур = запрет «адресных шаблонов». ψ требует, чтобы экстремальная плотность достигалась **без** внешней маркировки запрещённых копий.

### D8.2. Инвариант (стоимость запрещения)
\[ T_r(G):=K(G) - K(G\setminus \text{all }K_{r+1}) \]
— дополнительная стоимость кодирования запрета. **min‑K:** экстремалы — там, где \(T_r\) минимален без ярлыков.

### D8.3. Лемма (ярлык‑барьер)
Бить туроновскую границу без скрытой адресации запрещённых шаблонов невозможно (иначе \(\Delta K>0\)).

### D8.4. Фальсификация
- Семейство графов, бьющее границу Турана **без** адресации запрещённых копий и со **снижением** общей описательной длины.

### D8.5. Мини‑проверки
- Сравнить туроновские экстремалы с псевдослучайными конструкциями на предмет скрытых ярлыков; оценить \(T_r\).

### D8.6. Псевдокод
```
Input: family of graphs G_n
1) encode G_n without labels of forbidden subgraphs
2) estimate T_r(G_n) and edge counts
3) if beats Turán bound with ΔK=0 and K↓ → falsify; else accept
```
